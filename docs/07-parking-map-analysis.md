# PDF 车位图分析

## 概述

江南之星小区地下车库共有两层，原始资料为 CAD 导出的 PDF 图纸（图片型，非矢量）。

| 楼层 | 文件 | 说明 |
|------|------|------|
| 地下一层 | 地下一层车位图.pdf | 120×170cm 平面图 |
| 地下二层 | 江南之星地下室二层车位.pdf | 平面图 |

> ⚠️ **重要发现**：PDF 是 CAD 导出的图片格式（栅格图），AI 无法直接提取矢量车位数据。
> 所有车位数据必须手动录入。

### 区域划分

| 19# | 绿色 | ~25 | 顶部中间 |
| 20# | 粉色 | ~20 | 顶部左侧 |

**总计约 400+ 个车位**（具体数量需手动清点）

### 编号规律
- 格式：`区域号-序号`（如 `1-001`、`1-002`）
- 区域号与图中 `#` 号对应（如 1# 区域的车位编号为 `1-xxx`）
- 注意：没有 4#、6# 区域（可能是跳号或预留）

### 布局结构
- **通道**：白色区域，贯穿整个地下一层，连接各区域
- **柱子**：黑色小方块，分布在车位之间
- **出入口**：图的边缘，底部（近 1#/2#/3#）和顶部（近 19#/20#）

### 特殊标记
- 部分车位标注为**无障碍车位**（轮椅符号）
- 部分车位标注为**充电桩车位**（闪电符号）

## SVG 地图探索（已完成，暂缓）

### 历史尝试
- 使用 Python 脚本分析 PDF 图片，生成 SVG 车位图
- 生成了 V5 版本 SVG（244 个车位框），部署到测试页
- **结论**：AI 自动识别车位位置不够精确，生成的 SVG 与实际车位对不上

### 测试页面
- URL：https://parking.willeai.cn/svg-test.html（已下线）

### 为什么暂缓？
1. **精度不够**：AI 从图片识别车位位置，偏差较大
2. **优先级低**：MVP 阶段不需要地图，用户量 50+ 后再做
3. **成本高**：要完美复刻 CAD 图，需要人工标注或找物业要原始 CAD 文件

## 未来地图方案

### 方案 A：人工标注（推荐）
1. 找物业要 CAD 原始文件（矢量格式）
2. 用 CAD 软件导出 SVG
3. 手动调整车位编号和区域标签

### 方案 B：AI 辅助 + 人工校准
1. 用 AI 识别 PDF 图片，生成初始 SVG 框架
2. 人工逐个校准车位位置和编号
3. 工作量大但不依赖物业

### 方案 C：简化版（最轻量）
1. 不做精确车位图
2. 只做区域示意图（B1-A 在哪、B2-B 在哪）
3. 用户按区域找车位，不依赖精确位置

> **当前建议**：MVP 阶段用方案 C（区域示意图），用户量上来后再考虑方案 A。

## 数据录入清单

### 需要手动录入的数据

| 数据 | 数量 | 来源 |
|------|------|------|
| 楼栋列表 | ~30 幢 | 物业/业主 |
| 每幢单元数 | 1-2 个 | 物业/业主 |
| 地下一层车位 | ~400 个 | PDF 图纸 |
| 地下二层车位 | ~125 个 | PDF 图纸 |
| 车位所属区域 | 每个车位 | PDF 图纸 |
| 特殊车位标记 | ~10 个 | PDF 图纸 |

### 录入优先级

1. **先录入楼栋列表**（注册功能需要）
2. **再录入 5-10 个车位**（冷启动需要，不需要全部录完）
3. **后续边用边录**（业主绑定车位时自动录入）
