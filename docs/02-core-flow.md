# 核心流程设计

## 两条主线

```
出租方：车位空闲 → 发布共享 → 等人借用 → 确认 → 结束
借用方：需要车位 → 搜索可用 → 申请借用 → 被确认 → 停车 → 结束
```

## 出租方流程

### 首次使用
1. 微信打开小程序 / 网页
2. 手机号登录
3. 填写昵称、楼栋+单元（下拉选择）
4. 选择常用停车区域（A / B / C / D / E / F / G）
5. 绑定车位（输入车位号，如 E-001）
6. （可选）上传车位合同照片

### 日常发布
1. 打开首页 → 看到"我的车位"卡片
2. 点"发布共享"
3. 设置空闲时段（开始时间、结束时间）
4. 设置价格（默认 4 元/小时，20 元封顶）
5. 点确认 → 状态变为"共享中"
6. 等待有人借用

### 处理借用
1. 收到通知"有人要借你的车位"
2. 查看借用方信息（昵称、楼栋、车牌号）
3. 点"同意" → 借用开始
4. 点"📋 复制通知管家" → 一键生成文案，发给物业管家
5. 借用结束 → 自动结算

### 一键通知管家
借用确认后，出租方看到自动生成的文案：
```
您好，我是20幢2单元的业主张三。
我的车位 E-005 已借给邻居使用。
停车时间：今天 14:00 ~ 18:00
车牌号：浙A·12345
麻烦知悉，谢谢！
```
点"复制"按钮，粘贴发给管家即可。纯前端生成，无需后端改动。

## 借用方流程

### 首次使用
1. 微信打开小程序 / 网页
2. 手机号登录
3. 填写昵称、楼栋+单元（下拉选择）
4. 选择常用停车区域

### 找车位
1. 打开首页 → 看到"附近可借车位"
2. 默认显示常用区域附近的可借车位（最多 5 个）
3. 显示：车位号、价格、距离（区域）
4. 点击查看详情

### 借用流程
1. 选中一个车位 → 点"申请借用"
2. 输入车牌号
3. 确认借用时段
4. 提交 → 等待出租方确认
5. 确认后 → 开始计时
6. 停车结束 → 点"结束借用" → 自动计算费用

## 状态流转

### 车位状态
```
idle（空闲） → available（已发布共享） → occupied（被借用中） → available → ...
```

### 借用记录状态
```
pending（等待确认） → active（借用中） → done（已完成）
                   → disputed（有纠纷）
```

## 通知机制（MVP 阶段简化）

| 事件 | 通知谁 | 方式 |
|------|--------|------|
| 有人申请借用 | 出租方 | 微信模板消息 / 短信 |
| 借用被确认 | 借用方 | 微信模板消息 / 短信 |
| 借用即将到期 | 双方 | 微信模板消息 |

MVP 阶段：先用短信通知（已有阿里云短信接口），后续再加微信模板消息。
