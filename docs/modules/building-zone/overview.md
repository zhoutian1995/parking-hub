# 楼栋区域模块 概要设计

## 目标
提供小区楼栋-单元列表和停车区域列表，供注册和筛选使用。

## 范围
### 包含
- GET /api/buildings：返回楼栋-单元列表
- GET /api/zones：返回停车区域列表
- 管理后台维护楼栋数据

### 不包含
- 楼栋地理信息（经纬度）
- 楼栋-区域距离映射（用户自己选常用区域）

## 依赖模块
- database 模块（读 buildings、zones 表）
- 无业务逻辑依赖其他模块

## 数据来源
- zones 表：预填 7 条（A~G），基本不变
- buildings 表：手动录入（江南之星 30 幢楼），偶尔更新

## 核心流程
1. 前端注册时 → GET /api/buildings → 渲染楼栋下拉
2. 用户选楼栋 → 从该楼栋的 units 字段拆出单元列表 → 渲染单元下拉
3. 前端选区域 → GET /api/zones → 渲染区域选择
