# 前端模块 测试计划

## 测试范围
- 页面渲染和布局
- 登录流程
- 首页数据展示
- 发布共享流程
- 借用流程
- 一键通知管家

## 测试用例

| 编号 | 测试项 | 操作步骤 | 预期结果 | 实际结果 | 状态 |
|------|--------|----------|----------|----------|------|
| FE-001 | 首页加载（未登录） | 打开网站 | 显示登录按钮，不显示我的车位 | | |
| FE-002 | 首页加载（已登录） | 已登录状态打开 | 显示附近车位 + 我的车位 | | |
| FE-003 | 登录 | 输入手机号 → 登录 | 跳转完善信息页或首页 | | |
| FE-004 | 完善信息 | 选择楼栋 → 单元 → 区域 → 完成 | 跳转首页 | | |
| FE-005 | 发布共享 | 我的车位 → 发布 → 设置时段价格 → 确认 | 车位状态变为"共享中" | | |
| FE-006 | 申请借用 | 点击附近车位 → 输入车牌 → 申请 | 显示"等待确认" | | |
| FE-007 | 确认借用 | 收到申请 → 确认 | 车位状态变为"借用中" | | |
| FE-008 | 一键通知管家 | 确认借用后 → 点复制文案 | 剪贴板有正确文案 | | |
| FE-009 | 结束借用 | 借用中 → 结束 | 显示费用，车位恢复可用 | | |
| FE-010 | 手机端布局 | 用手机浏览器打开 | 布局正常，按钮可点击 | | |

```bash
open http://localhost:3000

# 3. 逐步测试每个页面和交互
```

## 验收标准
- [ ] 所有页面在手机端正常显示
- [ ] 登录→完善信息→首页 流程通顺
- [ ] 发布→被借→确认→结束 流程通顺
- [ ] 一键通知管家文案正确
- [ ] 空状态有友好提示（无记录、无车位）
