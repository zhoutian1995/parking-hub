# 新用户引导 + 空状态优化

## 模块概述

优化新用户体验，解决"第一次打开不知道怎么用"的问题。

## 目标用户

- 新注册用户（首次登录）
- 功能空状态（无车位、无记录、无搜索结果）

## 功能清单

### 1. 新用户引导流程

| 步骤 | 页面 | 引导内容 |
|------|------|----------|
| 1 | 登录后 | 弹窗：欢迎加入 ParkingHub！3步开始使用 |
| 2 | 完善信息 | 高亮楼栋选择器 + 提示"选择你的楼栋" |
| 3 | 首次进首页 | 高亮"绑定车位"按钮 + 提示"先绑定你的车位" |
| 4 | 绑定后 | 高亮"发布"按钮 + 提示"发布后邻居就能借用" |

**引导策略**：
- 仅展示一次（`localStorage` 记录 `has_seen_guide`）
- 可跳过（右上角"跳过"按钮）
- 非模态（不阻塞操作）

### 2. 空状态优化

| 场景 | 当前状态 | 优化后 |
|------|----------|--------|
| 未绑定车位 | "暂未绑定车位" | 插图 + "绑定车位后可发布共享" + 绑定按钮 |
| 无附近车位 | "登录后查看附近车位" | 插图 + "暂无可借车位，试试发布你的？" |
| 无借用记录 | "暂无记录" | 插图 + "还没有借用记录" + 浏览车位按钮 |
| 搜索无结果 | "未找到" | "未找到 A050，试试其他车位号？" + 热门推荐 |
| 无发布车位 | 发布页无选项 | "你还没有绑定车位，先去绑定吧" + 绑定按钮 |

### 3. 错误提示优化

| 错误类型 | 当前 | 优化后 |
|----------|------|--------|
| 网络错误 | `e.message` | "网络连接失败，请检查网络后重试" |
| 服务器错误 | 无 | "服务器开小差了，请稍后再试" |
| 车位号格式 | 无 | "车位号格式不正确，如 A050" |
| 重复绑定 | 无 | "该车位已被绑定" |
| 借用自己的车位 | 无 | "不能借用自己发布的车位" |
| 车位已被借用 | 无 | "该车位已被其他人借用" |

## 技术实现

### 前端

1. **引导组件**：`guide-overlay` div + CSS 动画
2. **空状态组件**：统一的 `empty-state` 组件（插图 + 文案 + 按钮）
3. **错误处理**：统一 `handleError()` 函数，分类处理不同错误

### 后端

1. 错误响应统一格式：`{ error: { code: 'XXX', message: 'xxx' } }`
2. 新增错误码：`SPOT_ALREADY_BOUND`, `BORROW_SELF_SPOT`, `BORROW_UNAVAILABLE`

## 验收标准

- [ ] 新用户首次登录看到引导流程
- [ ] 引导可跳过，且仅展示一次
- [ ] 所有空状态有插图+文案+操作按钮
- [ ] 网络错误、服务器错误有友好提示
- [ ] 业务错误有具体提示（如"车位已被绑定"）
