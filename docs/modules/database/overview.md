# 数据库模块 概要设计

## 目标
管理 ParkingHub 项目的所有数据存储，提供数据初始化、迁移和基础查询能力。

## 范围
### 包含
- SQLite 数据库初始化和连接管理
- 所有表的创建和结构定义
- 初始数据导入（车位数据、区域数据、楼栋数据）
- 数据库迁移脚本（加字段、改结构）

### 不包含
- 业务逻辑（由各业务模块处理）
- 数据查询优化（MVP 阶段不做）
- 数据备份策略（后续考虑）

## 输入
- 初始化脚本启动时自动执行
- 迁移脚本手动执行

## 输出
- 数据库文件：`database/parking.db`
- 连接对象供各 controller 使用

## 依赖模块
### 依赖谁
- 无（最底层模块，其他所有模块都依赖它）

### 被谁依赖
- auth 模块（读写 users 表）
- spot 模块（读写 spots 表）
- borrow 模块（读写 borrows 表）
- building-zone 模块（读写 buildings/zones 表）

## 核心流程
1. 应用启动 → 调用 `init.js` → 检查/创建数据库文件
2. 检查表是否存在 → 不存在则创建
3. 导出 `getDb()` 方法供其他模块使用
4. 迁移脚本独立运行 → 修改表结构/导入数据

## 技术选型
- **SQLite**（better-sqlite3）：轻量、无依赖、适合小规模项目
- 不选 MySQL/PostgreSQL：MVP 阶段不需要，部署简单
